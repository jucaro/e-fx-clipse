JavaFX 2.0 Tooling and Runtime for Eclipse and OSGi

Create a Eclipse-ViewPart Use JavaFX for Eclipse plugins

JavaFX allows you embed JavaFX scenes into SWT-Uis using FXCanvas. The following tutorial will guide you through the creation of an JavaFX enabled ViewPart.

1.

Download the JavaFX-SDKs (2.1 as of this writing) from Oracle.

Oracle decided to make it hard for you, not providing zip-Files for released versions. So you have 2 choices:

  • Use 2.1:
    Accept the license. Copy the link location and replace .exe with .zip.
    Hint the mac name is javafx_sdk-2_1_0-macosx-universal.zip and the one from linux-32 javafx_sdk-2_1_0-linux-i586.zip
  • Use the preview builds for 2.2:
    Simply follow the link at the top of the page below the main title

I think I don't have to mention how silly this policy is. See RT-18807 for an unconvincing explanation.

For this tutorial we'll use the 2.2 preview builds.

2.

File > New > Project ...

3.

Select "Plug-in Project" and hit the "Next" button

4.

Enter a project name e.g. MyFXViewPart and hit the "Next" button

5.

Check that "This plug-in will make contributions to the UI" is checked and hit the "Next" button

6.

Select the "JavaFX Eclipse 3.x ViewPart" template and hit the "Next" button

7.

Keep the default settings and hit the "Finish" button

8.

You'll be prompted if there's no repackaged JavaFX-Bundle yet found in your workspace. Hit the "Yes" button.

9.

Enter the bundle version you'd like to use for the repackaged plugin. Add the SDKs to repackage for the operation system / processor archtitectures you'd like to support. In the hit the "Finish" button.

10.

The result of the wizard are 2 OSGi-Bundles:

  • MyFXViewPart:

    The bundle with your ViewPart

  • javafx.osgi:

    This bundle holds the javafxrt.jar and dlls for the architectures you decided to deploy to.

    The bundle is a very special one because it does not export any packages but is a pure deployment unit for the JavaFX jar and its dlls.

    The wiring of those jars is done by an OSGi-Extension which is part of e(fx)clipse runtime modules - nothing you need to worry about.

11.

Implement your view by adding some JavaFX code

12.

Launch an "Inner Eclipse" by bringing up the context-menu on the "MyFXViewPart" project and select Run As > Eclipse Application

13.

In the launched instance bring up Quick Access (Ctrl+3) and enter "My Fx View" and select the presented choice.

14.

A view with your JavaFX content will be shown in the bottom

15.

Create a new feature using. New > Feature Project

16.

Enter a name for the feature e.g. "MyFXViewFeature" and hit the "Finish" button

17.

Open the "Plug-ins" tab or the feature editor and add the following plugins:

  • at.bestsolution.efxclipse.runtime.javafx
  • at.bestsolution.efxclipse.runtime.osgi
  • at.bestsolution.efxclipse.runtime.workbench3
  • javafx.osgi
  • MyFXViewPart

18.

Create an update site using New > Project ... > Plug-in Development / Update Site Project

19.

Enter a updatesite name e.g. "MyFXViewUpdate" and hit the "Finish" button

20.

Create a new category e.g. with ID "myfxview" and name "My JavaFX View" and add your "MyFXViewFeature" and hit the "Build All" button

21.

The result of the operation should look similar to this. You'll get the exported plugins and some p2 repository jars so that you can install it using the "Install New Software ..." wizard.
